3)  What is Babel? What is Webpack/Vite?
acts as a transpiler, converting modern JavaScript code (like ES6+ features and JSX used in React) into an older, backwards-compatible version of JavaScript (usually ES5) that can run on virtually all browsers

babel: You write code using the latest language features or special syntax like JSX, and Babel translates it into code that every browser understands, ensuring your application works everywhere.

vite: Webpack is like a powerful factory manager. It takes all the raw materials (your code and assets), processes them according to specific instructions (loaders and plugins), and packages them into streamlined boxes (bundles) for delivery to the browser.
Key features: Highly customizable, extensive plugin ecosystem, code splitting, and tree-shaking for large, complex applications.

Q) stateful vs stateless components:-
Stateful Components:
Definition: Maintain internal data (state) and can change over time.
Usage: Handle logic, user interactions, API calls, and complex data.
Syntax: Typically class components (older) or functional components using useState hooks.
Pros/Cons: Higher complexity, less reusable, but crucial for dynamic interaction. 
Stateless Components:
Definition: Do not maintain internal state; they only receive props to render UI.
Usage: Presentational UI (e.g., buttons, cards).
Syntax: Simple functional components.
Pros/Cons: Highly reusable, easy to test, and better performance

Q14. Lifting state up — 0/2
Ideal answer:
“Lifting state up” means moving shared state to the closest common ancestor so multiple siblings can read/update it.
Example:
Parent keeps selectedId and passes it down to List (to set selected) and Details (to show selected item).

Q15- Dependency array role — USEEFFECT QUESTION
Your answer: Mentions lifecycle and dependency array; slightly off.
Correction:
The dependency array tells React when to re-run the effect:

[] → run once on mount (cleanup on unmount).
[a, b] → run after first render and whenever a or b change.
(no array) → run after every render.
It doesn’t by itself encode “update” vs “unmount”—cleanup function handles unmount.
