MY NEW GOAL:-
A Solution Architect focuses on designing comprehensive solutions to meet specific business needs,while a Software Architect focuses on the internal structure and design of software systems.  Solution architects consider the entire system, including software, hardware, and infrastructure, while software architects focus on the software itself. 

FIRST TO BE SOFTWARE ARCHITECT:
Designs the high-level structure and behavior of software systems. 
Ensures that the software is well-designed, efficient, and scalable. 
Focuses on the internal components of a system, how they interact, and how they are assembled. 
May also be involved in code reviews and selecting appropriate technologies for the project. 

THEN SOLUTION ARCHITECT:-
Designs solutions that address business requirements, considering various factors like cost, performance, and scalability. 
Works at the intersection of business strategy and technology, ensuring that IT infrastructure aligns with business goals. 
May be involved in selecting technologies, evaluating vendors, and integrating different systems. 
Often acts as a consultant or advisor, helping businesses make strategic decisions about technology. 


In essence:
A Solution Architect thinks about the "big picture," ensuring that the solution meets the overall business needs. 
A Software Architect focuses on the "inner workings" of the software system, ensuring it's well-designed and efficient. 

Example:
Imagine a company wants to implement a new customer relationship management (CRM) system.
A Solution Architect would be responsible for designing the entire CRM solution, considering factors like:
Which CRM software to choose (e.g., Salesforce, Microsoft Dynamics). 
How the CRM system will integrate with existing databases and systems. 
The hardware and infrastructure required to support the CRM system. 
The business processes that will be affected by the CRM system. 
A Software Architect would be responsible for designing the architecture of the specific CRM software, including:
The database design. 
The user interface design. 
The algorithms and code that power the CRM system. 
How the different modules of the CRM system will interact with each other.

/*
 * Determine the original side lengths and return an array:
 * - The first element is the length of the shorter side
 * - The second element is the length of the longer side
 * 
 * Parameter(s):
 * literals: The tagged template literal's array of strings.
 * expressions: The tagged template literal's array of expression values (i.e., [area, perimeter]).
 */
function sides(literals, ...expressions) {
    let sqrVal = (expressions[1] * expressions[1])- 16 * expressions[0];
    let arr =[];
    // positive
    let s1= (expressions[1]+ Math.sqrt(sqrVal))/4;
    arr.push(s1);
    
    // negative
    let s2=(expressions[1]- Math.sqrt(sqrVal))/4;
    arr.push(s2)
    
    return arr.sort();
}


function main() {
    let s1 = +(readLine());
    let s2 = +(readLine());
    
    [s1, s2] = [s1, s2].sort();
    
    const [x, y] = sides`The area is: ${s1 * s2}.\nThe perimeter is: ${2 * (s1 + s2)}.`;
    
    console.log((s1 === x) ? s1 : -1);
    console.log((s2 === y) ? s2 : -1);
}
